# Purpose

This document defines the Value Objects used throughout the platform.

Unlike Entities, Value Objects do not have identity.

They represent immutable business concepts.

---

# What is a Value Object?

A Value Object is identified by its values rather than an ID.

Examples

Money

NPR 500

equals

NPR 500

They are the same value.

---

# Entity vs Value Object

| Entity | Value Object |
|----------|--------------|
| Has identity | No identity |
| Mutable | Immutable |
| Lifecycle | No lifecycle |
| Has ID | No ID |
| Example: Product | Example: Money |

---

# Design Principles

- Immutable
- Self-validating
- Equality by value
- No database identity
- Reusable across contexts

---

# Money

Purpose

Represents a monetary amount.

Properties

- amount
- currency

Examples

NPR 500

USD 100

Business Rules

- Cannot be null
- Currency required
- Amount cannot exceed configured precision

Used By

- Product Price
- Sale
- Payment
- Refund
- Purchase

# Domain Rules

Value Objects

- Cannot exist independently
- Cannot be queried directly
- Cannot have repositories
- Cannot publish events

They only exist inside Entities or Aggregates.
