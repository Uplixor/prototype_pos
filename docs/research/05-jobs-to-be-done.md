# Executive Summary

Customers do not purchase software because they want features.

They purchase software because they need to accomplish a job.

The purpose of this document is to identify those jobs so that future product decisions optimize for outcomes rather than feature count.

---

# Objectives

This document answers:

- Why do customers buy our platform?
- What jobs are they trying to complete?
- What frustrates them today?
- How should our platform improve their workflow?

# Research Findings

Customers purchase outcomes.

Not software.

Every feature must exist because it helps complete a job.

---

# Recommendations

Every feature proposal must answer:

1. Which job does this solve?

2. Whose job is it?

3. How is it solved today?

4. Why is that insufficient?

5. How does our solution improve it?

If a feature cannot answer these questions, it should not be built.
